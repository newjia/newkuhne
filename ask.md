 **项目名称**：销售订单智能查询 Agent

**核心目标**  
构建一个 Microsoft Copilot Studio 智能体（LLM 驱动），让销售厂家用户通过自然语言直接查询、分析和未来操作 SQLite 数据库中的订单表及所有销售相关表，实现快速、准确的业务洞察。

**必须满足的要求**

1. **输入**：用户任意自然语言描述（如“本月订单总额”“按客户分组的平均订单金额 Top 10”“把订单123改成已发货”）

2. **LLM 深度参与**  
   - 理解用户意图（含模糊、多步、组合需求）  
   - 动态选择、编排、调用合适的 Skills / MCP Tools  
   - 推理、校验、后处理结果  
   - 生成自然语言 + 结构化表格的友好回答

3. **当前核心能力**（读操作）  
   - SUM / AVG / COUNT / MIN / MAX  
   - GROUP BY（按客户/时间/产品/地区等）  
   - LIST / 明细查询 / 过滤 / 排序 / Top N  
   - 时间范围筛选（本月/去年Q4/自定义日期）  
   - 多条件组合查询

4. **数据接入方式**  
   - 通过 **MCP Server** 安全连接 SQLite 数据库  
   - 所有数据库操作封装为参数化、可复用的 Tools  
   - 智能体不直接写 SQL，全部靠调用 MCP Tools

5. **高可扩展性（最关键）**  
   - 新增任何功能（新表查询、新聚合、更新订单、插入记录、删除、发送邮件、导出 Excel 等）  
   - **只需**在 MCP Server 增加/修改一个 Tool  
   - Agent 侧无需或极少改动（自动动态发现新 Tool）  
   - 目标：业务人员/运维按模板“配一个 Skill”即可立即生效

**交付物**  
- 1 个 Copilot Studio Agent（已连接 MCP）  
- 1 个 SQLite 专用 MCP Server（支持动态添加 Tool）  
- 初始一套 Skills/Tools（覆盖订单读场景）  
- 简单文档/模板：如何在 MCP Server 新增 Tool 的操作指南

**这就是最终交付形态**：一个智能体 + 一个可随时扩展的 MCP 后端，LLM 全程驱动，用户说啥就能干啥，未来加功能只改 MCP 就行。
